# 替换 Script Service 为 theta_1-main 真实脚本

## 背景

当前 [script_service.py](langgraph_agent/backend/app/services/script_service.py) 中定义了 9 个**虚构脚本**（如 `01_data_upload_clean.sh`），这些脚本在仓库中并不存在。需要替换为 `theta_1-main/scripts/` 中的**真实脚本**，并按照 `scripts_usage_guide.md` 中的参数定义准确配置。

## 需要纳入的脚本（从"上传"开始，共 11 个）

按流水线顺序：

- **02_clean_data.sh** - 数据清洗（CSV/目录模式），参数: `--input`, `--language`, `--text_column`, `--label_columns`, `--keep_all`, `--preview`, `--min_words`
- **02_generate_embeddings.sh** - Embedding 生成（03 的子脚本，用于恢复），参数: `--dataset`, `--mode`, `--model_size`, `--batch_size`, `--epochs`, `--label_column`, `--exp_dir`, `--gpu`
- **03_prepare_data.sh** - 数据准备（All-in-one），参数: `--dataset`, `--model`, `--model_size`, `--mode`, `--vocab_size`, `--language`, `--bow-only`, `--check-only` 等
- **04_train_theta.sh** - THETA 训练，参数: `--dataset`, `--model_size`, `--mode`, `--num_topics`, `--epochs`, `--batch_size`, `--hidden_dim`, `--learning_rate`, `--kl_start/end/warmup`, `--patience`, `--gpu`, `--language`
- **05_train_baseline.sh** - 基线模型训练（11 种模型），参数: `--dataset`, `--models`, `--num_topics`, `--epochs`, `--batch_size`, `--gpu`, `--language`, `--with-viz`
- **06_visualize.sh** - 可视化，参数: `--dataset`, `--baseline`, `--model`, `--model_size`, `--mode`, `--language`, `--dpi`
- **07_evaluate.sh** - 评估，参数: `--dataset`, `--model`, `--num_topics`, `--model_size`, `--mode`
- **08_compare_models.sh** - 模型对比，参数: `--dataset`, `--models`, `--num_topics`, `--output`
- **10_quick_start_english.sh** - 英文快速开始，参数: dataset_name
- **11_quick_start_chinese.sh** - 中文快速开始，参数: dataset_name
- **12_train_multi_gpu.sh** - 多 GPU 训练，参数: `--dataset`, `--num_gpus`, `--model_size`, `--mode`

不纳入的脚本：`01_setup.sh`（一次性安装）、`09_download_from_hf.sh`（下载工具）、`13_test_agent.sh`（测试）、`14_start_agent_api.sh`（基础设施）、`run_*.sh`（批量/GPU 特化脚本）

## 变更文件

### 1. [langgraph_agent/backend/app/services/script_service.py](langgraph_agent/backend/app/services/script_service.py)

- 将 `SCRIPTS_DIR` 从硬编码 `/root/autodl-tmp/scripts` 改为通过 `settings` 配置读取，默认指向 `theta_1-main/scripts/`
- 替换 `AVAILABLE_SCRIPTS` 字典，按 `scripts_usage_guide.md` 中的参数定义，为每个脚本配置正确的 `id`、`name`、`description`、`parameters`、`category`
- 将 category 分为: `preprocessing`（02_clean_data, 02_generate_embeddings, 03_prepare_data）、`training`（04_train_theta, 05_train_baseline, 12_train_multi_gpu）、`visualization`（06_visualize）、`evaluation`（07_evaluate, 08_compare_models）、`quickstart`（10/11_quick_start）

### 2. [langgraph_agent/backend/app/api/scripts.py](langgraph_agent/backend/app/api/scripts.py)

- 更新便捷端点，将当前的虚构端点（`/train`, `/embedding`, `/evaluate`, `/visualize`, `/pipeline`）替换为匹配真实脚本的端点：
  - `POST /scripts/clean` -> 02_clean_data.sh
  - `POST /scripts/prepare` -> 03_prepare_data.sh
  - `POST /scripts/train-theta` -> 04_train_theta.sh
  - `POST /scripts/train-baseline` -> 05_train_baseline.sh
  - `POST /scripts/visualize` -> 06_visualize.sh
  - `POST /scripts/evaluate` -> 07_evaluate.sh
  - `POST /scripts/compare` -> 08_compare_models.sh
  - `POST /scripts/quick-start` -> 10/11_quick_start
- 更新各端点的 Pydantic Request 模型，匹配真实脚本参数

### 3. 前端（无需改动）

- [theta-frontend3/app/admin/monitor/page.tsx](theta-frontend3/app/admin/monitor/page.tsx) 只做 `GET /api/scripts` 健康检查，自动适配新脚本列表，无需修改
